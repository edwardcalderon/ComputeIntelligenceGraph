✓ vs ✓✓ — Estado de entrega
Un tick gris-lila (✓) significa que el mensaje fue enviado pero el bot aún no lo procesó. Dos ticks azul claro (✓✓) confirman que el bot recibió, procesó y está generando la respuesta.

Sparkline — Línea de tendencia
Gráfico de línea compacto para mostrar evolución temporal. Área bajo la curva en morado translúcido. El punto más alto lleva un círculo marcador. Eje X muestra días abreviados (L M X J V S D).

Gauge donut — % de cumplimiento
Arco semicircular. Pista gris-verde (#EAF3DE), relleno según nivel: verde ≥80%, ámbar 50–79%, rojo <50%. El porcentaje se muestra en el centro. Debajo: filas de contexto (logrado, días, ritmo).

6 · Timeline — trazabilidad de eventos y bloqueos

Punto morado — Evento de creación
Primer nodo del timeline. Punto morado = el sistema registró el inicio del proceso (pedido creado, solicitud generada). La línea vertical gris que cuelga conecta con el siguiente evento.
punto #6C3DE8 conector #e8e4f8 tipo: inicio
08:41 · Motor
Validación crédito
Punto azul — Proceso / Validación
Nodo de un paso de procesamiento interno (validación, consulta, cálculo). Punto azul (#378ADD), texto en azul oscuro. Indica que el sistema ejecutó una regla o consulta automática sin fallo.
punto #378ADD texto #0C447C tipo: proceso
08:41 · Regla
Crédito insuf.
Punto rojo — Fallo / Regla activada
Nodo de error. Punto rojo (#E24B4A), texto en rojo oscuro. Indica el momento exacto en que una regla de negocio no se cumplió y detuvo el flujo. Siempre incluye el código de la regla (ej. CAR-04 §2.3).
punto #E24B4A texto #791F1F tipo: fallo
08:42 · Notif.
Escalado Gerente
Punto ámbar — Acción pendiente / Escalado
Último nodo del timeline. Sin línea conectora debajo (es el final). Punto ámbar = hay una acción humana pendiente (aprobación, ajuste, notificación). El proceso continúa fuera del sistema automático.
punto #EF9F27 texto #854F0B tipo: pendiente

Bloque rojo al pie del timeline de trazabilidad. Resume en una sola línea: valor actual, valor requerido y la política que disparó el bloqueo. Es el único elemento que combina datos numéricos con referencia normativa.
fondo #FCEBEB texto #791F1F formato: dato · dato · POL §n
